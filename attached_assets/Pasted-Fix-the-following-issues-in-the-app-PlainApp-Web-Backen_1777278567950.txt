1. CRITICAL BUG FIX – GraphQL Errors

---

Problem:

- "Failed to coerce as ContactInput"
- "Failed to coerce as NoteInput"

Root cause:

- Frontend sends null values, empty fields, wrong types (int instead of string), or missing required fields.

Fix:

A. Sanitize ALL outgoing data before sending:

Rules:

- No null values → replace with empty string ""
- No undefined fields
- No empty arrays unless allowed
- Convert numeric enums to strings (e.g. phone type: 2 → "mobile")
- Trim all strings

Example fix:

Before sending:

- organization = null → ""
- label = "" → "default"
- type = 2 → "mobile"

B. Add universal sanitizer function:

JS/TS:
function sanitize(obj) {
return JSON.parse(JSON.stringify(obj, (key, value) => {
if (value === null || value === undefined) return "";
if (typeof value === "string") return value.trim();
return value;
}));
}

Use before every mutation:
sendMutation(sanitize(input))

C. Add try-catch to ALL mutations:

try {
await mutate(...)
} catch (e) {
showToast("Something went wrong, retrying...");
}

D. Backend tolerance (IMPORTANT):

- Make GraphQL schema accept optional fields
- Avoid strict coercion failure
- Add default values in resolvers
